# Модификации проекта

Это модифицированная версия проекта [bitrix-tools/env-docker](https://github.com/bitrix-tools/env-docker).

## Оригинальный проект

- **Репозиторий:** https://github.com/bitrix-tools/env-docker
- **Лицензия:** GPL-3.0
- **Автор:** 1С-Битрикс
- **Описание:** Контейнерное окружение для разработки на Битрикс

## Дата модификации

2025-12

## Внесенные изменения

### 1. Удобная разработка
- Настроено монтирование локальной директории `www/` вместо Docker-тома и добавлен bindfs-мост `www-bindfs/`
- Теперь можно безопасно редактировать файлы напрямую на хосте (через `www-bindfs/`), и изменения сразу видны в контейнерах
- Исходники лежат в `./www/`

### 2. Безопасность
- Добавлен `.gitignore` для исключения секретных файлов из Git
- Созданы шаблоны `.env_sql.example` и `.env_push.example` для безопасной настройки
- Секретные файлы (пароли БД, ключи) больше не попадут в репозиторий

### 3. Документация
- Добавлен файл `SETUP.md` с инструкциями по настройке окружения
- Обновлена документация в `www/README.md`
- Добавлен раздел про bindfs-монтаж `www/` и команды `scripts/mount-www-bindfs.sh` / `scripts/umount-www-bindfs.sh`

## Технические детали

### Изменения в docker-compose.yml
- Заменено `www_data:/opt/www/` на bindfs-монтирование `./www-bindfs:/opt/www/` в сервисах:
  - `php`
  - `nginx`
  - `cron`
- Добавлен алиас `dev.bx` и сервис `loopback`, обеспечивающие корректную проверку сокетов Bitrix

### Новые файлы
- `.gitignore` - исключает секретные файлы
- `.env_sql.example` - шаблон для паролей БД
- `.env_push.example` - шаблон для секретного ключа Push-сервера
- `SETUP.md` - инструкция по настройке
- `MODIFICATIONS.md` - этот файл
- `scripts/mount-www-bindfs.sh` / `scripts/umount-www-bindfs.sh` - автоматический монтаж каталога сайта через bindfs
